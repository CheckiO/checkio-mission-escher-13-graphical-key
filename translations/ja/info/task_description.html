正しい順番でボタンを押すと同時に急に足元の床が崩れ あなたは地下牢のより深い階層に落ちて行きました。 この城はまだあなたを驚かすのを止めてはいなかったのです! 幸運にも落下による怪我はありませんでした。 さもなければあなたは家に帰る必要がなくなっていたでしょう。

あなたは立ち上がって落ち着きを取り戻すと想像も付かない宝物を背後に隠しているかのようなドアを発見しました。 それは巨大かつ金属製で数世紀を経たにもかかわらずただ一点の錆もありません。 二つのことが考えられます。 「ステンレス製である」、 「何年にもわたって誰かが手入れをしてきた」 あなたは最初の考えが真実であって欲しいと思いました。 亡霊と出会うのは一回で充分ですから。

このドアには鍵穴に似た様なものさえありませんでした。 しかし突起をもつタッチスクリーンのような場所がありました。 あるパターンを描けば解錠できるロックに似ています。 どこからこの古城にそのようなテクノロジーがもたらされたのでしょうか? 無数のとりとめもない考えが浮かんでは消えましたが全ては帰還するまで保留できることです。今の優先事項はこのドアを開けることです。

スクリーンに触れると各点は数値に変化しました。 恐らく解錠のためのパターンを描く必要があるようです。 「例外...(Except...)」数値の選び方に何か法則性があるのでしょう?

あなたの仕事は数値の集合の 2次元配列 ( 第1引数 ) を受け取り、 左上角から右下角へたどる場合の数値の合計の最大値を返す関数を書くことです。 またその際 N個 ( 第2引数 ) より多くの点を訪れてはなりません。 それには開始点と終了点を含みます。 8方向 ( 縦, 横, 斜め )に移動可能です。

入力: 数値の配列, 経路の長さ ( 注: 開始点(左上角)と終了点(右下角)を含む点の総数 )

出力: 経路上の数値の合計 ( 最大値 )

例:

      [0, 4, 9, 5, 3],
      [7, 2, 5, 1, 4],
      [3, 3, 2, 2, 9],
      [2, 6, 1, 4, 0]], 9) == 46

g_key([[1, 6, 7, 2, 4],

       [2, 6, 1, 4, 0]], 9) == 46
{% endif %}

どのように使われるか: 経路探索に使用できます。

事前条件:
2x2 <= grid size <= 5x5
